📘 Documentación Técnica — CONY POS

Sistema de punto de venta para restaurante de comida rápida con delivery integrado.

⚙️ Stack Tecnológico

ComponenteTecnología
BackendPython 3.12 + FastAPI
Base de datosSQLite3 (archivo .db)
FrontendHTML5 + JavaScript + Bootstrap 5 (plano, sin framework)
AutenticaciónHTTP Basic Auth con bcrypt
Servidor webCaddy (proxy reverso + SSL)
Impresiónprint-client.py en Mac local → puerto 9100
WhatsAppOpenClaw + conybot

🚀 Deploy: De DEMO a CONY

⚠️ Siempre probar en DEMO antes de pasar a CONY.

# 1. Respaldar CONY
cp /var/www/pos+ia/clientes/cony/cony_rest/pos_cony.py /backup/pos_cony_$(date +%Y%m%d).py
cp /var/www/pos+ia/clientes/cony/cony_rest/cony_cony.db /backup/cony_cony_$(date +%Y%m%d).db

# 2. Copiar backend DEMO → CONY
cp /var/www/pos+ia/clientes/demo/backend/cony_rest/pos_cony.py \
   /var/www/pos+ia/clientes/cony/cony_rest/pos_cony.py

# 3. Migrar BD (solo esquema nuevo)
sqlite3 /var/www/pos+ia/clientes/cony/cony_rest/cony_cony.db "
ALTER TABLE productos ADD COLUMN es_ingrediente INTEGER DEFAULT 0;
ALTER TABLE productos ADD COLUMN stock REAL DEFAULT 0;
ALTER TABLE productos ADD COLUMN stock_minimo REAL DEFAULT 0;
ALTER TABLE productos ADD COLUMN unidad TEXT DEFAULT 'pieza';
ALTER TABLE productos ADD COLUMN descripcion TEXT DEFAULT '';
CREATE TABLE IF NOT EXISTS recetas (...);
"

# 4. Copiar frontend
cp /var/www/pos+ia/clientes/demo/frontend/frontend_demo/*.html \
   /var/www/pos+ia/clientes/cony/frontend/*.html

# 5. Reiniciar CONY
systemctl restart cony-pos.service

🔐 Roles de Usuario

RolPermisos
superuserAcceso total. Puede todo.

userTomar pedidos, ver dashboard. No edita productos.
repartidorVer pedidos asignados, cambiar estado a entregado

🖨️ Impresión

El sistema usa una cola SQL (impresoras_cola). El print-client.py corre en la Mac local del restaurante y consulta la cola periódicamente.

Impresoras:

  • Caja (.68) — Ticket para el cliente (puerto 9100)
  • Cocina (.69) — Comanda para cocina (puerto 9100)

📦 Webhooks Delivery

El sistema recibe pedidos de delivery a través de webhooks HTTP POST:

  • /api/delivery/recibir — Recibe pedido de app externa
  • /api/delivery/autorizar — Autoriza pedido pendiente
  • /api/delivery/asignar — Asigna repartidor
