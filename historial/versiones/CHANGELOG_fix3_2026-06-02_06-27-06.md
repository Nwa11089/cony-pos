## Cambios

### 1. Arqueo simplificado (solo billetes x denominación + monedas total)
**Antes:** Inputs para 0, , ,  (moneda por denominación) — demasiado lento
**Ahora:** 
- 💵 ,000, 00, 00, 00, 0, 0 (cantidad de billetes cada uno)
- 🪙 Monedas: UN solo campo numérico para el total en monedas (, , , 0, fracciones)

### 2. Cálculo de faltante/sobrante corregido
**Antes:** El frontend enviaba efectivo_reportado = arqueo - fondo. El backend lo comparaba contra efectivo_esperado.
**Ahora:** El frontend envía total_arqueado (el conteo físico). El backend calcula:
  - efectivo_reportado = total_arqueado - fondo_inicial (ventas efectivo generadas)
  - diferencia = efectivo_reportado - efectivo_ventas (lo que generó vs lo que debió generar)
El cajero nunca ve estos cálculos.

### 3. Dirección de cliente se actualiza siempre
**Antes:** Solo actualizaba dirección si el cliente tenía "Mostrador" guardado
**Ahora:** Si el usuario envía una dirección diferente a la registrada, se actualiza siempre

### 4. Validación de pedidos pendientes al cerrar corte
- Frontend: verifica antes de mostrar el modal de arqueo
- Backend: rechaza con mensaje claro si hay pedidos en estado pendiente o en-preparacion
- El cajero debe marcar todos los pedidos como LISTOS antes de cerrar
